## [1.0.0] — 2025-04-11

### Added
- Multi-user authentication with JWT + bcrypt (register, login, logout, `/api/auth/me`)
- Email verification flow — new accounts must verify before logging in
- Password reset with UUID tokens and 1-hour expiry
- Per-user world isolation — all world routes scoped to `req.user.id`
- World management: create, start, stop, restart, delete
- Per-world Docker containers using `jacobsmile/tmodloader1.4:latest`
- Automatic game port allocation from a configurable range (default `7777–7900`), stored in SQLite
- World settings: max players, password, world size, difficulty, seed, autosave interval, shutdown message
- Steam Workshop mod management — add by URL or ID, set server/client/both scope per mod
- Steam Workshop name lookup via the public `GetPublishedFileDetails` API
- Per-world playit.gg tunnel support — optional `SECRET_KEY` per world stored in settings
- Global playit.gg tunnel via Docker Compose service (installer-configured)
- Bootstrap admin account — created on first start from `ADMIN_EMAIL` + `ADMIN_TEMP_PASS`
- Admin panel: list all users and worlds, promote/demote roles, manually verify accounts, delete users
- Live log viewer — tails container logs with 4-second polling; detects `Server started` and `Creating world`
- World start overlay — shows live log stream while the server is initialising
- SQLite database (`better-sqlite3`) for users, sessions, and port allocations
- SMTP email delivery via `nodemailer` with support for any STARTTLS provider
- Single-page frontend (vanilla JS, no build step) with dark theme, world cards, settings form, mods table, admin tables
- Interactive bash installer with SMTP provider presets, playit.gg setup, and SMTP live test
- Three-mode uninstaller: full removal / keep data / containers only
- Optional NAS backup script (`backup.sh`) with rsync + cron job generated by installer
- Docker Compose stack with bind-mounted worlds, data volume, and Docker socket

### Security
- `JWT_SECRET` fatal startup check — server refuses to start if the variable is not set
- ISO-8601 regex validation on the `since` query parameter in `/api/worlds/:name/logs`
- Logs route rewritten to use `execFileSync` with array arguments — no shell interpolation
- `express-rate-limit` applied to `/api/auth/login` and `/api/auth/forgot` (20 req / 15 min / IP)
- World name validated against `^[a-zA-Z0-9_-]+$` at creation time
- Container names derived from user ID prefix + sanitised world name — no user input reaches the shell directly in `startWorld`
